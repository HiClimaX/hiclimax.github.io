# Team Members Data (Markdown Format)

# This file contains team member information in YAML-like blocks separated by ---.
# Each block represents one team member. Required fields are marked with (required).
#
# Field explanations:
# - name: Full name of the team member (required, string, e.g., "Dr. Jane Doe")
# - role: Job title or position (required, string, e.g., "Senior Researcher")
# - affiliation: Organization and location (required, string, format: "Institution · Department · Country")
# - category: Team section (required, one of: core, method, regional, engineering)
#   - core: Principal investigators and core leadership (3-5 people)
#   - method: Lead researchers for each downscaling method (4-6 people)
#   - regional: Regional coordinators for case study cities (3-4 people)
#   - engineering: Software engineers, data managers, community staff (4-5 people)
# - emoji: Unicode emoji for visual distinction (optional, defaults to 👤 if missing)
# - photo: Path to headshot photo (optional, relative path like "data/team/photos/name.jpg")
#   - If photo exists, avatar becomes clickable link to full image
#   - If missing or broken, falls back to emoji
#   - Recommended: 300-500px square, JPG/PNG/WebP
# - github: Full GitHub profile URL (optional, e.g., "https://github.com/username")
# - email: Email address (optional, e.g., "name@domain.com")
# - orcid: Full ORCID profile URL (optional, e.g., "https://orcid.org/0000-0000-0000-0000")
#
# Notes:
# - Empty optional fields will hide the corresponding social link
# - Use UTF-8 encoding to preserve emoji
# - Keep one member per block, separated by ---
# - Photos should be placed in data/team/photos/ folder
